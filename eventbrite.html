Eventbrite

The Tab Index issue was also found on Eventbrite website. Only on Google Chrome and Microsoft Edge that users are able to use Tab Index for navigation on the web, but on Firefox Quantum and Safari, Tab Index cannot detect objects of an entire page. It just jumps between the searching area of the very top of the page.

On the homepage, all category titles’ text colour does not meet the colour contrast ratio standard on background images. As well as in the inner page, the website occasionally uses white and thin text over the light background colour. This would be difficult for users who have vision conditions or colour blindness.

And also, a great number of images were missing alternative text, as well as call-to-action buttons existed without value or text in it. Lack of these information would be the main issue for screen reader users. They will be unable to reach fully information and experience from the website.

The First Heading (<h1>) is the meaning of the page title which is very important according to WebAIM standard, but some pages of Eventbrite are missing first heading tag. It was used for tracking navigation where the user is using screen reader. This will be difficult for users with the screen reader to check where they are on the website.

Furthermore, there are some links, which do not have any label or content inside them, which is lack of description for the icon that user can be misunderstood. This will be a critical issue for users with the screen reader where they may not understand what the purpose of these links are.

In addition, on the main page of event search by location, a location form has multiple labels, which should have at most one label for each form according to WebAIM standard. This would affect the screen reader performance as it may not read the label or read only one label.

Moreover, another critical issue is on responsive to small devices. Most navigations and features on the website were hidden when opening on mobile (width smaller than 991 pixels). Mobile users are unable to use the website with efficient and effectiveness like on a desktop. This could limit the user’s ability to reach for content on the website (Duddell, 2016). A hamburger menu icon without label may confuse users when they see it for the first time.

Nonetheless, the all these critical issues were solved by assistive tools. Instead of navigating through the website by Tab Index, Mouse Keys allowed the user to use their numeric keypad control their cursor. They will easily navigate all feature and achieve full experience from the website like using a mouse or trackpad. Users can also use ATbar to increase their accessibility and readability on the website. There are many useful features on ATbar that users are able to use to adjust the colour contrast to fit their vision, such as adjusting text size, text colour, and background colour. However, there are some issues that need to solve in HTML, such as images alternative text, missing buttons value, lack of icon description, multiple form labels, and navigation menu on responsive design. In addition, if multiple labels of a form are necessary, it can be used aria-labelled by attribute as second labels.

Eventbrite received an average score of 72% on Web2Access.org.

The evaluation was completed on Firefox Quantum version 57.0.3 (64-bit), Safari version 11.0.2, Google Chrome version 63.0.3239.84 (64-bit), and Microsoft Edge version 5.105860.0 on Windows 10 by Parallels Desktop (virtual machine) in MacBook Air (Early 2014) with macOS High Sierra (10.13.2) and VoiceOver as a screen reader.
