This Website

There is no critical accessibility issue found on this site because it was built following accessibility guideline and use HTML5 and ARIA attribute to improve the website accessibility.

As well as, an order of structure was simple which is easy to follow which is understandable on no styles website with obviously contrasting heading size. This would boost users' learnability and can learn how to use the website at first visit. To which be able to understand what screen reader (VoiceOver on macOS) was read aloud of an entire website. And it allowed user to zoom up to 300 persent with readable contents and correct structure.

Moreover, this website is completely designed for support responsive. Contents were properly arranging on both desktop and mobile devices, without any omit or hide contents which will not affect user experience when they use crossing devices.

However, some common issues still found on the web. A link colour has lower colour contrast ratio than the accessibility standard requires.

And another concern is main navigation on mobile device, it was using scrolling navigation style which some user who does not have experiance with it before may misunderstand and do not know how to use it.

The evaluation was completed on Firefox Quantum version 57.0.3 (64-bit), Safari version 11.0.2, Google Chrome version 63.0.3239.84 (64-bit), and Microsoft Edge version 5.105860.0 on Windows 10 by Parallels Desktop (virtual machine) in MacBook Air (Early 2014) with macOS High Sierra (10.13.2) and VoiceOver as a screen reader.

This website is a part of coursework for Designing Usable and Accessible Technologies (COMP 6219).
University of Southampton.
